# Project Management

## Priority Matrix
| | Urgent | Not Urgent |
|---|---|---|
| **Important** | DO NOW (bugs, security) | SCHEDULE (features, refactor) |
| **Not Important** | DELEGATE (minor fixes) | ELIMINATE (nice-to-have) |

## Task Complexity
- S: < 1 file, simple change
- M: 2-3 files, moderate logic
- L: 4-10 files, new feature
- XL: 10+ files, system redesign -> break down thanh S/M

## Sprint Planning (Solo Dev)
1. Chon 1 feature chinh / sprint (1-2 tuan)
2. Max 3 tasks in-progress
3. Hoan thanh > bat dau moi
4. Demo/test cuoi sprint

## Risk Assessment
Truoc khi implement feature lon:
1. Dependencies: can lib/service nao?
2. Migration: breaking changes?
3. Testing: test the nao?
4. Rollback: revert duoc khong?
